    <article class="markdown-body entry-content" itemprop="mainContentOfPage"><h1><a id="user-content-high-quality-classes-homework" class="anchor" href="#high-quality-classes-homework" aria-hidden="true"><span class="octicon octicon-link"></span></a>High-quality Classes Homework</h1>

<h2><a id="user-content-task-1-good-abstraction" class="anchor" href="#task-1-good-abstraction" aria-hidden="true"><span class="octicon octicon-link"></span></a>Task 1. Good Abstraction</h2>

<ul>
<li>  Take the VS solution <code>Abstraction</code> and refactor its code to provide <strong>good abstraction</strong>.

<ul>
<li>  Move the properties and methods from the class <code>Figure</code> to their correct place.</li>
<li>  Move the common methods to the base class's interface.</li>
<li>  Remove all <strong>duplicated code</strong> (properties / methods / other code).</li>
</ul></li>
</ul>

<h2><a id="user-content-task-2-good-encapsulation" class="anchor" href="#task-2-good-encapsulation" aria-hidden="true"><span class="octicon octicon-link"></span></a>Task 2. Good Encapsulation</h2>

<ul>
<li>  Establish <strong>good encapsulation</strong> in the classes from the VS solution <code>Abstraction</code>.

<ul>
<li>  Ensure that incorrect values cannot be assigned in the internal state of the classes.</li>
</ul></li>
</ul>

<h2><a id="user-content-task-3-cohesion-and-coupling" class="anchor" href="#task-3-cohesion-and-coupling" aria-hidden="true"><span class="octicon octicon-link"></span></a>Task 3. Cohesion and Coupling</h2>

<ul>
<li>  Take the VS solution <code>Cohesion-and-Coupling</code> and refactor its code to follow the principles of <strong>good abstraction</strong>, <strong>loose coupling</strong> and <strong>strong cohesion</strong>.

<ul>
<li>  Split the class <code>Utils</code> to other classes that have strong cohesion and are loosely coupled internally.</li>
</ul></li>
</ul>

<h2><a id="user-content-task-4-inheritance-and-polymorphism" class="anchor" href="#task-4-inheritance-and-polymorphism" aria-hidden="true"><span class="octicon octicon-link"></span></a>Task 4. Inheritance and Polymorphism</h2>

<ul>
<li>  Redesign the classes and refactor the code from the solution <code>Inheritance-and-Polymorphism</code> to follow the best practices in high-quality classes.

<ul>
<li>  Extract abstract base class and move all common properties in it.</li>
<li>  Encapsulate the fields and make sure required fields are not left without a value.</li>
<li>  Reuse the repeating code though base methods.</li>
</ul></li>
</ul>
